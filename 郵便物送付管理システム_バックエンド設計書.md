## 1. DB設計

### ER図（概要）

```
staffs ────────────────────────────────┐
                                       │ created_by / sent_by
offices（事業所）                      │
   │                                   │
   │ 多対多                            │
   │                                   ↓
users（利用者）          mail_sends（送付レコード）
   │                         │
   └── user_offices ──────────┘
       （中間テーブル）       │
                              └── mail_send_batches（送付バッチ）
```

### テーブル定義

#### `staffs`

| カラム名 | 型 | 制約 | 説明 |
|---|---|---|---|
| id | BIGINT | PK | |
| name | VARCHAR(100) | NOT NULL | 氏名 |
| email | VARCHAR(255) | UNIQUE / NOT NULL | ログインID |
| password_hash | VARCHAR(255) | NOT NULL | |
| role | ENUM('ADMIN','STAFF') | NOT NULL | |
| is_active | BOOLEAN | DEFAULT true | |
| force_password_change | BOOLEAN | DEFAULT true | 初回変更フラグ |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

#### `users`

| カラム名 | 型 | 制約 | 説明 |
|---|---|---|---|
| id | BIGINT | PK | |
| name | VARCHAR(100) | NOT NULL | 氏名 |
| name_kana | VARCHAR(100) | | ふりがな |
| birth_date | DATE | | 生年月日 |
| notes | TEXT | | 備考 |
| is_active | BOOLEAN | DEFAULT true | |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

#### `offices`

| カラム名 | 型 | 制約 | 説明 |
|---|---|---|---|
| id | BIGINT | PK | |
| name | VARCHAR(200) | NOT NULL | 事業所名 |
| postal_code | VARCHAR(8) | | 郵便番号 |
| address | VARCHAR(500) | | 住所 |
| phone | VARCHAR(20) | | 電話番号 |
| is_active | BOOLEAN | DEFAULT true | |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

#### `user_offices`

| カラム名 | 型 | 制約 | 説明 |
|---|---|---|---|
| id | BIGINT | PK | |
| user_id | BIGINT | FK → users.id | |
| office_id | BIGINT | FK → offices.id | |
| created_at | TIMESTAMP | NOT NULL | |

> `user_id` + `office_id` にUNIQUE制約

#### `mail_send_batches`

| カラム名 | 型 | 制約 | 説明 |
|---|---|---|---|
| id | BIGINT | PK | |
| sent_by | BIGINT | FK → staffs.id | 送付処理者 |
| sent_at | TIMESTAMP | NOT NULL | 送付日時 |
| notes | TEXT | | メモ（任意） |
| created_at | TIMESTAMP | NOT NULL | |

#### `mail_sends`

| カラム名 | 型 | 制約 | 説明 |
|---|---|---|---|
| id | BIGINT | PK | |
| user_id | BIGINT | FK → users.id / NOT NULL | |
| office_id | BIGINT | FK → offices.id / NOT NULL | |
| send_type | ENUM('PLAN','MONITORING') | NOT NULL | |
| send_month | DATE | NOT NULL | 月初日で保存 |
| status | ENUM('PENDING','SENT','DONE') | DEFAULT 'PENDING' | |
| batch_id | BIGINT | FK → mail_send_batches.id / NULL可 | |
| created_by | BIGINT | FK → staffs.id | |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

### ステータス遷移

```
PENDING（送付待ち） → SENT（送付済み） → DONE（完了）
```

### 補足

- `send_month` は月初日で統一保存（例: 2025年5月 → `2025-05-01`）
- 月遅れ判定：`send_month < 当月の月初日 AND status = 'PENDING'`
- 重複防止：`user_id + office_id + send_type + send_month` の組み合わせはアプリ側でバリデーション

### SecurityConfig（概要）

```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/api/auth/login").permitAll()
            .requestMatchers("/api/staffs/**").hasRole("ADMIN")
            .requestMatchers("/api/**").authenticated()
            .anyRequest().permitAll()           // Reactの静的ファイル
        )
        .formLogin(form -> form
            .loginProcessingUrl("/api/auth/login")
            .successHandler(/* 200 JSON返却 */)
            .failureHandler(/* 401 JSON返却 */)
        )
        .logout(logout -> logout
            .logoutUrl("/api/auth/logout")
            .logoutSuccessHandler(/* 200 JSON返却 */)
        )
        .sessionManagement(session -> session
            .sessionCreationPolicy(SessionCreationPolicy.IF_REQUIRED)
        );
    return http.build();
}
```

## 4. DB初期化設計（schema.sql / data.sql）

### 方針

Flywayは使用せず、Spring Bootの標準機能である `schema.sql` / `data.sql` でDB初期化を行う。
環境ごとにDDLが異なる（MySQL vs PostgreSQL）ため、platformファイルで管理する。

### ファイル構成

```
src/main/resources/
  ├── schema-mysql.sql        ← 開発・テスト用DDL（MySQL / H2 MySQL互換モード）
  ├── schema-postgresql.sql   ← 本番用DDL（Heroku PostgreSQL）
  └── data.sql                ← 初期データ（共通）
```

### schema-mysql.sql（開発・テスト用）

```sql
-- スタッフ
CREATE TABLE IF NOT EXISTS staffs (
    id                    BIGINT       NOT NULL AUTO_INCREMENT PRIMARY KEY,
    name                  VARCHAR(100) NOT NULL,
    email                 VARCHAR(255) NOT NULL UNIQUE,
    password_hash         VARCHAR(255) NOT NULL,
    role                  ENUM('ADMIN','STAFF') NOT NULL DEFAULT 'STAFF',
    is_active             BOOLEAN      NOT NULL DEFAULT TRUE,
    force_password_change BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at            DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at            DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 利用者
CREATE TABLE IF NOT EXISTS users (
    id          BIGINT       NOT NULL AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(100) NOT NULL,
    name_kana   VARCHAR(100),
    birth_date  DATE,
    notes       TEXT,
    is_active   BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 事業所
CREATE TABLE IF NOT EXISTS offices (
    id          BIGINT       NOT NULL AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(200) NOT NULL,
    postal_code VARCHAR(8),
    address     VARCHAR(500),
    phone       VARCHAR(20),
    is_active   BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 利用者↔事業所紐付け
CREATE TABLE IF NOT EXISTS user_offices (
    id          BIGINT   NOT NULL AUTO_INCREMENT PRIMARY KEY,
    user_id     BIGINT   NOT NULL,
    office_id   BIGINT   NOT NULL,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uq_user_offices (user_id, office_id),
    FOREIGN KEY (user_id)   REFERENCES users(id),
    FOREIGN KEY (office_id) REFERENCES offices(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 送付バッチ
CREATE TABLE IF NOT EXISTS mail_send_batches (
    id         BIGINT   NOT NULL AUTO_INCREMENT PRIMARY KEY,
    sent_by    BIGINT   NOT NULL,
    sent_at    DATETIME NOT NULL,
    notes      TEXT,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (sent_by) REFERENCES staffs(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 送付レコード
CREATE TABLE IF NOT EXISTS mail_sends (
    id          BIGINT   NOT NULL AUTO_INCREMENT PRIMARY KEY,
    user_id     BIGINT   NOT NULL,
    office_id   BIGINT   NOT NULL,
    send_type   ENUM('PLAN','MONITORING')      NOT NULL,
    send_month  DATE                           NOT NULL,
    status      ENUM('PENDING','SENT','DONE')  NOT NULL DEFAULT 'PENDING',
    batch_id    BIGINT,
    created_by  BIGINT   NOT NULL,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id)    REFERENCES users(id),
    FOREIGN KEY (office_id)  REFERENCES offices(id),
    FOREIGN KEY (batch_id)   REFERENCES mail_send_batches(id),
    FOREIGN KEY (created_by) REFERENCES staffs(id),
    INDEX idx_status     (status),
    INDEX idx_send_month (send_month),
    INDEX idx_batch_id   (batch_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### schema-postgresql.sql（本番用）

```sql
DO $$ BEGIN
    CREATE TYPE staff_role  AS ENUM ('ADMIN','STAFF');
    CREATE TYPE send_type   AS ENUM ('PLAN','MONITORING');
    CREATE TYPE send_status AS ENUM ('PENDING','SENT','DONE');
EXCEPTION WHEN duplicate_object THEN NULL;
END $$;

CREATE TABLE IF NOT EXISTS staffs (
    id                    BIGSERIAL    PRIMARY KEY,
    name                  VARCHAR(100) NOT NULL,
    email                 VARCHAR(255) NOT NULL UNIQUE,
    password_hash         VARCHAR(255) NOT NULL,
    role                  staff_role   NOT NULL DEFAULT 'STAFF',
    is_active             BOOLEAN      NOT NULL DEFAULT TRUE,
    force_password_change BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at            TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at            TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS users (
    id          BIGSERIAL    PRIMARY KEY,
    name        VARCHAR(100) NOT NULL,
    name_kana   VARCHAR(100),
    birth_date  DATE,
    notes       TEXT,
    is_active   BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at  TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS offices (
    id          BIGSERIAL    PRIMARY KEY,
    name        VARCHAR(200) NOT NULL,
    postal_code VARCHAR(8),
    address     VARCHAR(500),
    phone       VARCHAR(20),
    is_active   BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at  TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS user_offices (
    id          BIGSERIAL NOT NULL PRIMARY KEY,
    user_id     BIGINT    NOT NULL REFERENCES users(id),
    office_id   BIGINT    NOT NULL REFERENCES offices(id),
    created_at  TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uq_user_offices UNIQUE (user_id, office_id)
);

CREATE TABLE IF NOT EXISTS mail_send_batches (
    id         BIGSERIAL PRIMARY KEY,
    sent_by    BIGINT    NOT NULL REFERENCES staffs(id),
    sent_at    TIMESTAMP NOT NULL,
    notes      TEXT,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS mail_sends (
    id          BIGSERIAL   PRIMARY KEY,
    user_id     BIGINT      NOT NULL REFERENCES users(id),
    office_id   BIGINT      NOT NULL REFERENCES offices(id),
    send_type   send_type   NOT NULL,
    send_month  DATE        NOT NULL,
    status      send_status NOT NULL DEFAULT 'PENDING',
    batch_id    BIGINT      REFERENCES mail_send_batches(id),
    created_by  BIGINT      NOT NULL REFERENCES staffs(id),
    created_at  TIMESTAMP   NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP   NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX IF NOT EXISTS idx_status     ON mail_sends(status);
CREATE INDEX IF NOT EXISTS idx_send_month ON mail_sends(send_month);
CREATE INDEX IF NOT EXISTS idx_batch_id   ON mail_sends(batch_id);
```

### data.sql（初期データ・共通）

```sql
-- 初期管理者（パスワード: "changeme" のBCryptハッシュ・初回ログイン時に変更必須）
INSERT INTO staffs (name, email, password_hash, role, is_active, force_password_change)
SELECT '管理者', 'admin@example.com',
    '$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy',
    'ADMIN', TRUE, TRUE
WHERE NOT EXISTS (
    SELECT 1 FROM staffs WHERE email = 'admin@example.com'
);
```

> `WHERE NOT EXISTS` で2回目の起動時の重複INSERTを防ぐ。

### テーブル変更時の運用ルール

| ケース | 対応 |
|---|---|
| テーブル構造を変更したい | schema-mysql.sql と schema-postgresql.sql を両方修正 |
| ローカルDBをリセット | MySQLでDB DROP → CREATE → アプリ再起動で再作成 |
| 本番DBを変更したい | Heroku上でSQL手動実行（`heroku pg:psql`） |

## 5. テスト環境設計（バックエンド）

### 5.1 テスト戦略（テストピラミッド）

```
          ▲
         /E2E\          少数・重要フローのみ
        /------\        （Playwright）
       /  統合  \       APIレイヤー・DB結合
      /----------\      （Spring Boot Test）
     /  ユニット  \     ビジネスロジックの検証
    /--------------\    （JUnit5 + Mockito）
```

### 5.2 バックエンド テスト

#### テストライブラリ

| ライブラリ | 用途 |
|---|---|
| JUnit5 | テストフレームワーク |
| Mockito | モック・スタブ |
| Spring Boot Test | 統合テスト（@SpringBootTest） |
| MockMvc | Controllerテスト |
| Testcontainers | テスト用PostgreSQL（本番と同じDB） |
| AssertJ | アサーション（流暢なAPI） |

#### レイヤー別テスト方針

| レイヤー | テスト種別 | 使用ツール | DB |
|---|---|---|---|
| Service | ユニットテスト | JUnit5 + Mockito | 不要（モック） |
| Repository | 統合テスト | @DataJpaTest | H2 or Testcontainers |
| Controller | MVCテスト | MockMvc | モック（Serviceをモック） |
| E2E（API全体） | 統合テスト | @SpringBootTest | Testcontainers |

#### ディレクトリ構成


#### テストコード例（Serviceユニットテスト）

```java
@ExtendWith(MockitoExtension.class)
class MailSendBatchServiceTest {

    @Mock MailSendRepository mailSendRepository;
    @Mock MailSendBatchRepository batchRepository;
    @InjectMocks MailSendBatchService service;

    @Test
    void まとめて送付済みにする_正常系() {
        // given
        var ids = List.of(1L, 2L, 3L);
        var request = new CreateMailSendBatchRequest(ids, "5月定例送付");
        var mockSends = ids.stream()
            .map(id -> { var ms = new MailSend(); ms.setId(id); return ms; })
            .toList();
        when(mailSendRepository.findAllById(ids)).thenReturn(mockSends);

        // when
        service.createBatch(request, 1L);

        // then
        verify(batchRepository).save(any(MailSendBatch.class));
        verify(mailSendRepository).saveAll(argThat(sends ->
            sends.stream().allMatch(s -> s.getStatus() == SendStatus.SENT)
        ));
    }

    @Test
    void 存在しないIDを指定した場合_例外をスロー() {
        when(mailSendRepository.findAllById(any())).thenReturn(List.of());
        assertThatThrownBy(() -> service.createBatch(...))
            .isInstanceOf(ResourceNotFoundException.class);
    }
}
```
